Framework WebThings - 1.3.9 release notes

R-1.3.9-20060517 Date: 17.05.2006

Améliorations depuis 1.2

  • [1.3.9] menu: tag permettant d'activer / désactiver [node] dans un menu: menuActivateNode.
  • [1.3.9] menu: tag permettant d'exclure un [node] dans un menu popup: menuExcludeNode.
  • [1.3.9] template "process": le bouton "Ok" n'apparaît plus lorsqu'un process a été lancé. Possibilité néanmoins de le remettre en modifiant la variable "showProcessButton". Egalement possibilité de le renommer via la variable "okButtonLabel".
  • [1.3.8] action de menu: support d'un responseTarget "_blank";
  • [1.3.7] amélioration drastique des performances du tag "menu popup", d'un facteur 4 à 5;
  • [1.3.7] amélioration de l'urlStack.jsp;
  • [1.3.6] tag menu: warning dans les logs en cas de menu non trouvé;
  • [1.3.6] templates: propreté HTML;
  • [1.3.6] template process: la méthode du formulaire est mise à "post", comme pour les autres templates;
  • [1.3.5] /common et templates: propreté HTML;
  • [1.3.5] taglib dyntable: ajout de paramètres pour contrôler la présence des boutons ajouter et modifier;
  • [1.3.4] tmpl: capage et process: ajout de menu.js;
  • [1.3.2] menu: prise en compte des menus inactifs ("grisés");
  • [1.3.2] améliorations diverses des pages "common";
  • [1.3.1] réimplémentation de la gestion des boutons dans les pages (suppresion des balises "active=xxx");
  • [1.3.1] ajout de FWChooserTag.getSelectLink(). Exemple d'utilisation dans la FAQ R&T;
  • [1.3.1] dans tous les templates, ajout de <meta> pour décourager les browsers à mettre les pages en cache;
  • [1.3.1] suppression des balises <download> dans list et select.

Bugs corrigés depuis 1.2

  • [1.3.7] corrections dans menu.css;
  • [1.3.3] Corrections dans les scripts de gestion des boutons.
  • [1.3.1] Les custom tags HTML gèrent la possibilité de valeurs nulles dans les propriétés des vBeans.

Dépendances

  • Framework Jade: compilé avec framework_2-1-37
